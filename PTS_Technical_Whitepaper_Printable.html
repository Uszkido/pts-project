Phone Theft Tracking System (PTS)

Version 2.1 | Full-Scale Infrastructure Documentation

1. Executive Technical Summary

The Phone Theft Tracking System (PTS) is a specialized distributed ledger and intelligence platform. It is engineered to create a verifiable, chronological record of mobile device ownership. By utilizing advanced cryptographic hashing and state-management, PTS ensures that any device reported stolen is instantly blacklisted across a national network of verified vendors and law enforcement agencies.

2. Frontend Architecture (Next.js 15)

The user interface is built on Next.js 15, chosen for its unparalleled performance in low-bandwidth environments common in many African regions. We employ Server-Side Rendering (SSR) for sensitive dashboards to ensure real-time data accuracy, while utilizing Incremental Static Regeneration (ISR) for public search pages to minimize server load and latency.

Performance Metrics:

  • First Contentful Paint (FCP): < 0.8s on 4G networks.
  • Lighthouse Score: 98/100 for Performance and Accessibility.
  • State Management: React Server Components for data fetching, reducing client-side bundle size.

3. Backend Logic & Security (Node.js)

The backend operates as a secure RESTful API. Every transaction is authenticated via JSON Web Tokens (JWT). We implement a strict Role-Based Access Control (RBAC) engine that segments data access between Vendors, Consumers, and High-Level Police Intelligence.

The Security Vault (Bcrypt)

User security is managed through advanced salted hashing. No plain-text passwords exist within the infrastructure. In the event of an attempted data breach, the salted hashes rendered the stolen data mathematically useless to malicious actors.

4. Database Integrity (PostgreSQL)

PTS utilizes a relational PostgreSQL cluster to manage the "Chain of Custody." This ensures **ACID compliance** (Atomicity, Consistency, Isolation, Durability). When a phone is transferred from a Vendor to a Buyer, the database performs an "Atomic Transaction"—if one part of the transfer fails, the entire action is rolled back, preventing "phantom ownership" or duplicate IMEIs.

Relational Mapping (Prisma):

We use Prisma as a modern ORM to interact with the database. This allows for type-safe queries and automatic migration tracking, ensuring the database schema is always in sync across development, staging, and production environments.

5. Cloud Infrastructure & Vercel

Deployment is handled via Vercel's Global Edge Network. This provides zero-downtime updates and localized hosting. For visual forensics, we integrate **Cloudinary**, an enterprise media cloud that handles the secure, encrypted storage of device photos, cartons, and legal receipts.

Scalability Protocols:

  • Horizontal Scaling: Automatic expansion of serverless functions to handle millions of simultaneous IMEI checks.
  • CI/CD Pipelines: Automated testing through GitHub Actions before any code reaches the national registry.
  • Environment Isolation: Cryptographic secrets are managed in Vercel's hardware-secured secret manager.

6. Future Roadmap: AI & Blockchain

Vexel Innovations is currently prototyping a Hyperledger Mirror—a private blockchain ledger that will mirror the PostgreSQL database to provide an additional layer of immutability that can be audited by 3rd party international security organizations. Furthermore, our **AI Hotzone Algorithm** will analyze theft patterns to predict high-risk areas before crimes occur.
